# ソフトウェアの設計

設計については doc/design.md を参照せよ

# ディレクトリ構成

- test/: テストコード
- src/: axtのソースコード
- doc/: ドキュメント
- amalgamate.sh: 全ソースを単一のaxt.cに統合するスクリプト

# コマンド

- `make`: axtをビルドする
- `make test`: テストコードをビルドして実行する
- `make format`: コードをフォーマットする
- `make check-format`: コードがフォーマットされているかチェックする
- `make lint`: コードを静的解析する
- `make clean`: ビルド成果物を削除する
- `make amalgamate`: 全ソースを単一のaxt.cに統合する
- `make coverage`: カバレッジを測定する

# 開発の流れ

ファイルの変更が完了したら、`make test`を実行して全テストが正常に完了することを確認する。コードレビューのサブエージェントをすべて呼び出してコードレビューを行うこと。
コードレビュー結果に応じて、必要に応じてコードを修正し、再度コードレビューを行うこと。
全てのコードレビューエージェントから承認が得られるまで、このプロセスを繰り返すこと。

その後、ドキュメントの変更を担当するコードレビューエージェントを呼び出して、ドキュメントの変更を行うこと。
